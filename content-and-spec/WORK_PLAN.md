# תוכנית עבודה — בניית PDF לחיץ למסלולי שיקום מקצועי

## 1. מה למדתי ממסמכי המקור

### מטרת התוצר

התוצר הוא מסמך מידע דיגיטלי בפורמט PDF לחיץ עבור משתקמים שאושרה להם זכאות לשיקום מקצועי. המטרה היא לאפשר למשתקם להכיר את מסלולי השיקום, להבין את התמונה הרחבה, ולהגיע לפגישה עם עובד/ת השיקום מוכן יותר.

המסמך אינו מחליף את התהליך האישי מול עובד/ת השיקום, אלא תומך בו.

### עקרון חוויית המשתמש

העיקרון המרכזי הוא **הצגה מדורגת**: המשתמש לא רואה את כל התוכן בבת אחת, אלא בוחר כיוון, נכנס אליו, ורק אז נחשף לפירוט. לכן יש לבנות את ה-PDF כמו אתר קטן עם עמודים פנימיים וקישורים פנימיים.

### מבנה המידע

המסמך יכלול 11 עמודים:

1. עמוד הבית.
2. אבחון והתאמת תוכנית השיקום.
3. תוכניות טרום שיקום — עמוד ביניים.
4. סדנת העצמה.
5. סדנת השתלבות בעבודה.
6. תוכניות לחיזוק מיומנויות והכנה.
7. תוכניות ייחודיות לאוכלוסיות מיוחדות.
8. לימודים והכשרה — עמוד ביניים.
9. הכשרה מקצועית.
10. השכלה גבוהה.
11. השמה בעבודה.

### דרישות ניווט

בכל עמוד שאינו עמוד הבית צריכים להופיע:

- קישור חזרה לעמוד הבית.
- קישור חזרה לעמוד ההורה.
- סימון מיקום / breadcrumb, למשל: `עמוד הבית ← לימודים והכשרה ← השכלה גבוהה`.

ההמלצה במסמך המקור היא לבנות **PDF אחד יחיד**, שבו המעבר נעשה בקישורים פנימיים בין עמודים / אזורים בתוך אותו קובץ.

## 2. החלטות מוצר ועיצוב

### קובץ יחיד, עם שתי גרסאות פלט

נשמור על עקרון המקור: המסמך יהיה יחידה אחת עם קישורים פנימיים. במקביל נשמר את תשתית הפרויקט הקיימת שמייצרת שתי גרסאות:

- `desktop` — A4 מרובה עמודים, מתאים למחשב ולהדפסה.
- `mobile` — עמוד ארוך, מתאים לקריאה בטלפון.

כלומר, מבחינת המשתמש יהיו שני קבצי PDF אפשריים, אך כל אחד מהם יהיה עצמאי ולחיץ בתוכו.

### שפה עיצובית

נשתמש באותה שפה עיצובית שכבר נשמרה בפרויקט:

- RTL מלא.
- פונטים Rubik / Heebo.
- Hero כחול עם גרדיאנט ואייקון.
- כרטיסים לבנים עם מסגרת כחולה עדינה.
- כרטיסי אפשרויות עם אייקונים.
- תיבות מידע רכות (`callout blue`).
- תיבות שים לב (`callout amber`) רק כשיש צורך.
- כרטיסי תוכן מדורגים (`acc`, `cond`, `twoup`, `checks`, `important`).

### התאמת המבנה לעיצוב

המסמך החדש שונה ממדריכי הזכאויות הישנים: הוא לא מדריך ליניארי רגיל, אלא מסמך ניווטי. לכן יש להוסיף רכיבי עיצוב ייעודיים:

- כרטיסי ניווט גדולים בעמוד הבית.
- כרטיסי ניווט לעמודי ביניים.
- פס breadcrumb בראש עמודים פנימיים.
- כפתורי חזרה: לבית ולעמוד ההורה.
- עוגנים פנימיים לכל אחד מ-11 העמודים.

## 3. תוכנית יישום טכנית

### שלב א — הכנת מודל נתונים פנימי

ניצור בתוך builder ייעודי מבנה נתונים שמייצג את 11 העמודים:

- `id` — מזהה פנימי באנגלית לקישור, למשל `home`, `diagnosis`, `pre_rehab`.
- `title` — כותרת העמוד בעברית.
- `subtitle` — כותרת משנה אם קיימת.
- `parent` — מזהה עמוד ההורה.
- `breadcrumb` — רשימת כותרות לנתיב המיקום.
- `kind` — `home`, `hub`, או `content`.
- `children` — עמודי המשנה, אם קיימים.
- `body` — תוכן הטקסט.

בשלב הראשון אין צורך לבנות parser מלא לקובצי הטקסט. אפשר להזין את הנתונים ידנית מתוך `02_content.txt.txt`, כדי לשמור על שליטה בעיצוב ובחלוקת הכרטיסים.

### שלב ב — יצירת builder חדש

ניצור קובץ חדש:

```text
pdf-generator/pages/rehab_options.py
```

תפקידו:

1. לקרוא את `base.css` ואת הפונטים המקומיים.
2. לבנות HTML אחד שמכיל את כל 11 העמודים.
3. להוסיף לכל עמוד `id` פנימי לצורך קישורים.
4. להשתמש באותם helper components מהתבנית, עם הרחבה לרכיבי ניווט.
5. לכתוב HTML ביניים ל־`pdf-generator/_html/rehab-options.html`.

### שלב ג — הרחבת CSS לרכיבי ניווט

נוסיף ל־`pdf-generator/base.css` רכיבים ייעודיים למסמך החדש:

- `.guide-page` — מעטפת עמוד פנימי.
- `.nav-grid` — רשת כרטיסי ניווט.
- `.nav-card` — כרטיס מסלול לחיץ.
- `.breadcrumb-line` — סימון מיקום.
- `.page-actions` — אזור כפתורי חזרה.
- `.back-link` — כפתור חזרה לבית / להורה.
- `.pdf-anchor-page` — הגדרת שבירת עמוד בדסקטופ בין עמודי התוכן.

בדסקטופ כל אחד מ-11 העמודים צריך להתחיל בעמוד A4 חדש ככל האפשר. במובייל אין לבצע שבירות מלאכותיות שיחתכו את הרצף.

### שלב ד — קישורים פנימיים

נשתמש בקישורי HTML רגילים:

```html
<a href="#diagnosis">אבחון והתאמת תוכנית השיקום</a>
```

וכל עמוד יקבל:

```html
<section id="diagnosis">...</section>
```

יש לבדוק ב-PDF שהקישורים נשמרים לאחר הרינדור דרך Chromium/Playwright.

### שלב ה — עדכון build.sh

נוסיף יעד חדש:

```bash
./build.sh rehab-options
```

ונעדכן את המפות:

```bash
SLUG[rehab-options]=rehab-options
BUILDER[rehab-options]=pages/rehab_options.py
```

נשאיר את יעד `template` כרפרנס עיצובי.

### שלב ו — יצירת PDF ובדיקה ידנית

נבנה:

```bash
cd pdf-generator
./build.sh rehab-options
```

תוצרים צפויים:

```text
pdf/desktop/rehab-options.pdf
pdf/mobile/rehab-options.pdf
```

נבדוק:

- שכל 11 העמודים קיימים.
- כרטיסי עמוד הבית מובילים לעמודים הנכונים.
- עמודי ביניים מובילים לילדים שלהם.
- כפתור חזרה לבית עובד מכל עמוד.
- כפתור חזרה להורה עובד מעמודי רמה 3.
- breadcrumb נכון בכל עמוד.
- הדסקטופ קריא ומחולק נכון ל-A4.
- המובייל קריא כעמוד ארוך ולא נחתך.
- הפונטים, הצבעים והכרטיסים דומים לרפרנסים ב־`pdf/reference/`.

## 4. תוכנית תוכן לפי עמוד

### עמוד 1 — בית

- Hero: `אושר לך שיקום מקצועי`.
- כותרת משנה: `מהן האפשרויות העומדות בפניך?`.
- פסקת פתיחה.
- ארבעה כרטיסי מסלול לחיצים:
  - אבחון והתאמת תוכנית השיקום.
  - תוכניות טרום שיקום.
  - לימודים והכשרה.
  - השמה בעבודה.
- פסקת סיום בתיבה רכה.

### עמוד 2 — אבחון והתאמת תוכנית השיקום

עמוד תוכן ישיר. מתאים למבנה של כרטיסי מידע מדורגים:

- מתי משתמשים באבחון.
- מי מבצע את האבחון.
- מה כולל התהליך.
- שיחות אישיות והיבטים רפואיים/תפקודיים.
- סיכום והמלצות.

### עמוד 3 — תוכניות טרום שיקום

עמוד ביניים. יכלול:

- הסבר קצר על תפקיד תוכניות טרום השיקום.
- הבחנה בין סדנאות מרכזיות לתוכניות ייחודיות.
- ארבעה כרטיסי ניווט לילדים.

### עמודים 4–7 — תוכן תחת טרום שיקום

- סדנת העצמה.
- סדנת השתלבות בעבודה.
- תוכניות לחיזוק מיומנויות והכנה.
- תוכניות ייחודיות לאוכלוסיות מיוחדות.

בעמוד התוכניות הייחודיות כדאי להשתמש בכרטיסי משנה לפי אוכלוסייה: בריאות הנפש, לקויות תקשורת, מחלימים מסרטן, עיוורון ולקות ראייה, יוצאות/יוצאים ממעגל הזנות, ליקויי שמיעה, נפגעי ראש.

### עמוד 8 — לימודים והכשרה

עמוד ביניים. יכלול:

- הסבר קצר על התאמה ללימודים אקדמיים או להכשרה מקצועית.
- שני כרטיסי ניווט:
  - הכשרה מקצועית.
  - השכלה גבוהה.

### עמודים 9–10 — הכשרה מקצועית והשכלה גבוהה

שני עמודי תוכן נפרדים, עם מבנה דומה:

- למי המסלול מתאים.
- תנאי אישור מרכזיים.
- מהו המסלול.
- דוגמאות תחומים.

### עמוד 11 — השמה בעבודה

עמוד תוכן ישיר. כדאי להציג כתהליך:

- מטרת תוכנית ההשמה.
- התאמה אישית.
- שלב היכרות ובירור צרכים.
- חיבור לשוק העבודה.
- סיוע מעשי.
- ליווי לאחר קבלה לעבודה.

## 5. סדר עבודה מומלץ

1. להוסיף CSS לרכיבי הניווט החדשים.
2. ליצור את `pages/rehab_options.py` עם מבנה 11 העמודים.
3. להוסיף יעד `rehab-options` ל־`build.sh`.
4. לבנות HTML בלבד ולבדוק אותו בדפדפן אם אפשר.
5. לבנות PDF דסקטופ ומובייל.
6. לבדוק קישורים פנימיים.
7. להשוות ויזואלית מול `pdf/reference/desktop-style-reference.pdf` ו־`pdf/reference/mobile-style-reference.pdf`.
8. לבצע ליטוש ריווחים, כותרות ושבירות עמוד.
9. לעדכן README עם הוראות build למסמך החדש.
10. לבצע commit ו-PR.

## 6. סיכונים ונקודות לבדיקה

- קישורי anchor פנימיים ב-PDF עשויים להתנהג שונה בין מציגי PDF שונים; חובה לבדוק לפחות בדפדפן ובמציג PDF רגיל.
- בדסקטופ, אם מכריחים כל עמוד תוכן להתחיל ב-A4 חדש, יכול להיות שייווצרו עמודים עם הרבה שטח לבן. צריך לאזן בין תחושת "עמודים" לבין ניצול מקום.
- במובייל אסור להשתמש בשבירות עמוד קשיחות שעלולות ליצור חיתוך או גלילה לא טבעית.
- עמוד "תוכניות ייחודיות לאוכלוסיות מיוחדות" ארוך יחסית; ייתכן שיידרש פיצול פנימי לכרטיסי משנה כדי למנוע עומס.
- יש לשמור על נוסח התוכן כפי שהוא, אך מותר לחלק אותו ויזואלית לכרטיסים ופסקאות לפי המבנה.

pdf-generator/pages/
