# מחולל PDF למדריכי שיקום

מחולל זה נשמר כבסיס נקי ליצירת מדריכי PDF חדשים למבוטחים. הוא שומר את השפה העיצובית שכבר אושרה: RTL, פונטים מקומיים, Hero כחול, כרטיסי מידע לבנים, רכיבי `בקצרה`, תיבות `שים לב`, אייקונים, ושתי גרסאות פלט — דסקטופ ומובייל.

## מה נשאר בפרויקט

```text
pdf-generator/
├── base.css              # מערכת העיצוב המשותפת לכל מדריך
├── render.js             # HTML → PDF באמצעות Chromium
├── build.sh              # סקריפט בנייה
├── pages/
│   └── template.py       # תבנית עיצוב נקייה לתוכן עתידי
├── fonts/                # פונטים מקומיים Rubik / Heebo
├── package.json          # playwright-core
└── _html/                # HTML ביניים שנוצר בזמן build ואינו נשמר

pdf/reference/
├── desktop-style-reference.pdf  # רפרנס חזותי לגרסת A4 / דסקטופ
└── mobile-style-reference.pdf   # רפרנס חזותי לגרסת מובייל ארוכה

content-and-spec/
├── 01-structure-and-navigation.txt  # מבנה, ניווט ומטרה למסמך החדש
├── 02_content.txt.txt               # תוכן מלא לפי עמודים
├── README.md                        # סיכום תפקיד התיקייה
└── WORK_PLAN.md                     # תוכנית עבודה ליישום ה-PDF הלחיץ
```

תיקיות וקבצים של התוכן הישן — קובצי Markdown ישנים, רוב קובצי ה־PDF המוכנים, וקוד האתר המקורי — הוסרו כדי להשאיר כאן רק את מה שנחוץ ליצירת המדריכים החדשים. נשארו שני קובצי PDF בלבד כרפרנס חזותי: דוגמת דסקטופ ודוגמת מובייל.

## עקרונות העיצוב שנשמרו

- **שתי גרסאות PDF לכל מדריך**:
  - `desktop` — עמודי A4, חדים ונוחים לצפייה במחשב ולהדפסה.
  - `mobile` — עמוד אחד ארוך ורציף, מותאם לקריאה בטלפון.
- **פונטים מקומיים**: Rubik / Heebo, ללא תלות ברשת.
- **RTL מלא**.
- **Hero כחול** עם גרדיאנט, אייקון, עיגולים דקורטיביים וכותרת לבנה.
- **כרטיסי תוכן לבנים** עם מסגרת כחולה עדינה וצל עדין.
- **רכיבים חוזרים**: `summary`, `eligbar`, `acc`, `cond`, `callout`, `twoup`, `checks`, `important`.
- **שבירות עמוד בדסקטופ בלבד** כדי למנוע חיתוך כרטיסים ולשמור על מובייל כגלילה רציפה.

## רפרנסים חזותיים שנשארו

כדי שיהיה קל להשוות את המסמכים החדשים לעיצוב שאושר, נשמרו שני קבצים בלבד מתוך התוצרים הישנים:

- `pdf/reference/desktop-style-reference.pdf` — דוגמת A4 / דסקטופ.
- `pdf/reference/mobile-style-reference.pdf` — דוגמת מובייל ארוכה.

הקבצים האלה מיועדים לרפרנס עיצובי בלבד, ולא כמקור תוכן למדריכים החדשים.

## בניית תבנית עיצוב לבדיקה

```bash
cd pdf-generator
npm install
./build.sh template
```

הפקודה תיצור:

```text
pdf/desktop/template-style-reference.pdf
pdf/mobile/template-style-reference.pdf
```

## מקור התוכן החדש

חומר המקור למסמך הבא נמצא בתיקיית `content-and-spec/`, שהחליפה את השם `pdf-source`. התיקייה מפרידה בין מפרט המבנה והניווט לבין תוכן הטקסט המלא, כדי שנוכל לבנות PDF לחיץ אחד עם היררכיית עמודים פנימית. תוכנית היישום המפורטת נמצאת ב־`content-and-spec/WORK_PLAN.md`.

## הוספת מדריך חדש בהמשך

1. מעתיקים את `pages/template.py` לקובץ חדש, למשל `pages/rehab-plan-options.py`.
2. מחליפים את תוכן ה־Hero, ה־summary והסקשנים בתוכן החדש.
3. שומרים על אותם helper components כדי לשמר את השפה העיצובית.
4. מוסיפים מזהה חדש ל־`SLUG` ול־`BUILDER` בתוך `build.sh`.
5. מריצים `./build.sh <id>` ובודקים את שתי הגרסאות.
